Backing Up

It is a good idea to back-up your game on a regular basis; bad things do happen.

The file you need to back up is db.sqlite3, and is in the QuestJS root directory.

You need to decide for yourself how often you need to back-up. If you are working eight hours a day on your game, a daily back-up is advised. If you just spend a few hours a week, a monthly may be more reasonable. Think about how much effort you will have lost if you hard drive dies just before your next back-up, and compare that to the effort of doing each back-up.

You should save the file on another computer. An easy way to do that is to e-mail it to yourself, or to attach it to a draft e-mail.

Restoring

Restoring your game is a simple matter of installing QuestJS, and then copying the db.sqlite3 file into the QuestJS root directory.

Notes

This process also allows you to move your game to a new computer. Just install the editor on the new computer, and transfer across the db.sqlite3 file. I would guess this would work even when moving between operating systems, say from MacOS to Linux, as long as the db.sqlite3 file is formatted the same across both.
